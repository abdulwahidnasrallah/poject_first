# إجابات أسئلة المناقشة

## 1. لماذا نستخدم `concat` بدلًا من `push` عند تحديث مصفوفة في الحالة؟

نستخدم `concat` لأنه يُرجع مصفوفة جديدة بدون تعديل المصفوفة الأصلية
(immutable)، وهذا مهم في React لأن تحديث الحالة يجب أن يكون غير مباشر
لضمان إعادة التصيير (re-render).\
أما `push` فيقوم بتعديل المصفوفة الأصلية مباشرة (mutable)، وهذا قد يمنع
React من اكتشاف التغيير.

------------------------------------------------------------------------

## 2. لماذا نحتاج `[...votes]` عند التصويت بدلًا من تعديل المصفوفة مباشرة؟

نستخدم `[...votes]` لإنشاء نسخة جديدة من المصفوفة قبل التعديل، بدلًا من
تغيير المصفوفة الأصلية.\
هذا يحافظ على مبدأ **عدم التغيير المباشر (immutability)**، ويساعد React
على اكتشاف التغييرات وتحديث الواجهة بشكل صحيح.

------------------------------------------------------------------------

## 3. ما الفرق بين `onClick={{handleClick}}` و `onClick={{handleClick()}}`؟

-   `onClick={{handleClick}}`: تمرير الدالة نفسها ليتم تنفيذها عند
    الضغط.
-   `onClick={{handleClick()}}`: يتم تنفيذ الدالة مباشرة أثناء عملية الـ
    render وليس عند الضغط.

الصحيح غالبًا هو استخدام:

``` jsx
onClick={handleClick}
```

------------------------------------------------------------------------

## 4. متى نستخدم حالة واحدة معقدة (كائن) ومتى نستخدم عدة حالات منفصلة؟

-   نستخدم **حالة واحدة (كائن)** عندما تكون البيانات مترابطة مع بعضها
    (مثل بيانات نموذج form).
-   نستخدم **عدة حالات منفصلة** عندما تكون القيم مستقلة عن بعضها، مما
    يسهل القراءة والتعديل.

------------------------------------------------------------------------

## الحد الأدنى المطلوب (Minimum Goals)

-   فهم مفهوم **immutability** في React.
-   معرفة الفرق بين `push` و `concat`.
-   إتقان استخدام spread operator (`...`).
-   التمييز بين تمرير الدالة وتنفيذها في الأحداث.
-   اختيار الطريقة المناسبة لإدارة الحالة (state structure).
